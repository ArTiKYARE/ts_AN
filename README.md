# AnimeStream - Веб-сервис для просмотра аниме

## 📖 Описание
Современная платформа для просмотра аниме с минималистичным дизайном, поддержкой нескольких источников видео и озвучек, системой комментариев и админ-панелью.

## 🏗 Архитектура

│   │   ├── lib/           # Утилиты, API клиенты

## 🛠 Технологический стек

### Frontend
- **Next.js 14+** (App Router, Server Components)
- **React 18+** с хуками
- **TailwindCSS** + **shadcn/ui** для UI компонентов
- **Zustand** для управления состоянием
- **React Query** для кэширования API запросов
- **NextAuth.js** для аутентификации

### Backend
- **NestJS** (модульная архитектура)
- **Prisma ORM** для работы с БД
- **PostgreSQL** основная база данных
- **Redis** для кэширования и сессий
- **JWT** токены для аутентификации
- **class-validator** для валидации данных

### Инфраструктура
- **Docker** + **docker-compose** для контейнеризации
- **Nginx** как reverse proxy
- **Prometheus** + **Grafana** для мониторинга (опционально)

### Установка

1. Клонировать репозиторий:
```bash
git clone https://github.com/yourusername/animestream.git
cd animestream
```

2. Настроить переменные окружения:
```bash
cp .env.example .env
# Отредактируйте .env файл с вашими настройками
```

3. Запустить через Docker:
```bash
docker-compose up -d
```

4. Инициализировать базу данных:
```bash
docker-compose exec api npx prisma migrate dev
docker-compose exec api npx prisma db seed
```

5. Открыть в браузере:
- Фронтенд: http://localhost:3000
- API: http://localhost:4000
- Админ-панель: http://localhost:3001

## 📊 Схема базы данных

Основные сущности:
- **User** - пользователи с ролями (user, moderator, admin)
- **Anime** - информация об аниме (название, описание, жанры, год, статус)
- **Episode** - эпизоды с источниками видео
- **VideoSource** - источники видео (плееры, ссылки)
- **Dubbing** - варианты озвучки
- **Comment** - комментарии с вложенностью
- **Favorite** - избранное пользователей
- **WatchHistory** - история просмотров
- **ApiKey** - ключи для внешних API

## 🔌 Внешние API

Сервис использует адаптер паттерн для интеграции с внешними источниками данных:
- **AniList API** - метаданные аниме
- **Kitsu API** - альтернативный источник метаданных
- **Custom Aggregator** - собственный парсер

Все внешние данные кэшируются в Redis с настраиваемым TTL.
